    Data Collection

    Comment Section [Webnovel]

    In order to maintain anonymity, unless deliberately modified- the user handles of each comment is a type of animal within Primal Samsara. If that handle is changed by the user, or if it isn't, that data is stored in a Cloudflare Worker's KV along with the written comment.

    This is- generally necessary, in order to display the comments. Another analytic this collects internally is the amount of comments there are across the site.

    Cloudflare Web Analytics
    Unique information (Important)

    In terms of geography (location), the analytics collects country and region. While they cloudflare states that this is not personally identifiable information (PII) such as IP Addresses, I think there's generally an important concern missing here.

    While this isn't personally identifiable, if the site has very few visitors, it might be possible for me to guess the countries of specific people.

    If you're concerned about that, which is valid, I'd recommend a VPN. This should become less of a potential privacy issue as the site gains more visitors.

    As for other analytics, from what I've seen the cloudflare insights collects information about referrers (Like if a user came from Instagram, bing), device (Desktop, mobile), bandwidth (Amount of files being downloaded)

    The url used is also stored for analytics. (e.g. https://en.predation.jp/FAQ). This has always remained a confusing point as some visitors will bounce to the site from the preview development url which I swear is not indexed.

    The worst contender of these analytics is page views/sessions, unique visitors. Most of it is polluted by me just testing the site to see if certain features work, or some of it might be polluted by bots. This is why I've given up on using it as a useful analytic for webnovel views or reads.

    Site performance information

    Cloudflare collects Core Web Vital's data, like Largest Contentful Paint (LCP), Interaction to Next Paint (INP), First Input Delay (FID), Cumulative Layout Shift (CLS). Related to these, the most important analytic I'm looking at is Page Load Time.

    Some of the worst load times I've seen have been the site taking 10 whole seconds to load. I am genuinely really sorry to whoever had to deal with that. Part of this is related to the lack of a caching policy so that new updates to the webnovel aren't delayed by differences in caches.
